## Priority Scoring

### Severity Levels
| Level | Score | Description |
|-------|-------|-------------|
| 🔴 Critical | 4 | Immediate exploitation risk, data breach, service down |
| 🟠 High | 3 | Significant security risk, likely to cause issues |
| 🟡 Medium | 2 | Moderate risk, should be addressed |
| 🟢 Low | 1 | Minor issue, fix when convenient |

### Effort Levels
| Level | Score | Description |
|-------|-------|-------------|
| Trivial | 1 | < 1 hour, simple fix |
| Low | 2 | 1-4 hours |
| Medium | 3 | 1-2 days |
| High | 4 | 3-5 days |
| Complex | 5 | 1+ weeks |

### Priority Formula
```
Priority Score = Severity × (6 - Effort)
Higher score = Higher priority
```
